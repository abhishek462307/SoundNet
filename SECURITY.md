# Security Policy

## Reporting a vulnerability

If you discover a security issue, please do not open a public issue with exploit details.

Instead, report it privately to the project maintainers through the repository contact channel or private disclosure method used by the project.

Please include:

- a description of the issue
- affected endpoints or components
- reproduction steps
- potential impact
- any suggested remediation

## Scope

Security reports are especially relevant for:

- authentication and authorization
- agent messaging
- MCP server execution and trust controls
- secrets handling
- SSRF / unsafe remote execution risks
- multi-tenant or isolation concerns

## Response goal

We aim to acknowledge reports promptly and coordinate a responsible fix before public disclosure.
