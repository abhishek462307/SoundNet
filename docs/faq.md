# Sound Net FAQ

## Is Sound Net an AI model?
No. Sound Net is infrastructure, not a model.

## What does SOUND NET mean?
Service Orchestration Unified Network for Discovery, Navigation, Execution, and Transport.

## Is this an MCP server?
Not exactly. Sound Net can connect to MCP servers, import their tools, and manage them, but it is broader than a single MCP server.

## What is the main use case?
The main wedge is unified discovery, ranking, and execution of tools across HTTP APIs and MCP providers.

## Why does it also include agents and messaging?
Because the longer-term vision is a backend for multi-agent systems, not just a tool router.

## Is it production-ready?
It has many production-oriented pieces, but still needs real deployment hardening such as auth strategy, secrets management, multi-tenant isolation, monitoring, and operational setup.

## Does it support PostgreSQL?
Yes. It supports both file-backed development storage and PostgreSQL.

## Does it have a UI?
No, not currently.
